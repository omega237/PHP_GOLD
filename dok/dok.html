Installation

Zur Installation sollte die Verzeichnisstruktur unverändert in den PHP-Pfad oder einen Ordner in dem Projekt kopiert werden.

Erklärung der Verzeichnisstruktur

gold/ 					... Wurzelverzeichnis der Plausibilitätsprüfung
	diverses/ 			... enthält Dateien für den GOLD-Parser-Builder
	dok/ 				... Dokumentation für die Plausibilitätsprüfung
	Fassade/			... Auf den Anwendungsfall der Plausibilitätsprüfung abgestimmte Fassade für die Parser- und Interpreterbenutzung
	Interpreter/ 			... Interpreter für die Plausibilitätssprache
	Konstanten/ 			... PHP-Dateien mit vom Parser benutzten Konstanten
	Parser/ 			... der Parser auf Basis der GOLD-Parser-Engine
	Tabellen/ 			... PHP-Dateien mit vom Parser benutzten Tabellen
	Test/ 				... Testdateien für die Erklärung der Benutzung

Benutzung

Durch die Fassade werden Methoden zur Verfügung gestellt, die die Benutzung des Plausibilitätsprüfers vereinfachen. Der Ordner 

Test
enthält eine HTML-Datei 
plausi-einfach.html
eine JavaScript-Datei 
plausi.js
und eine PHP-Datei 
test.php

Mit diesen Dateien ist nachvollziehbar gemacht, wie die Plausibilitätsprüfung benutzt wird.

Referenz

Programmaufbau

Ein Programm besteht aus mehreren Zeilen, die jeweils eine Anweisung enthalten, die sich zu einem Wahrheitswert reduzieren lässt. Das Programm wird von oben nach unten und von links nach rechts ausgeführt. Syntaxfehler müssen vor der Ausführung behoben werden. Eine Zeile besteht aus beliebigen Kombinationen von Werten (Zahlen, Zeichenketten, Variablen) und Operatoren (s.u.) und einer optionalen Fehlerbehandlung, die an die Zeile angehängt wird und es erlaubt, bei einem falschen Ausdruck einen konstanten Text auszugeben.
Um einen Ausdruck so aufzubauen, daß er zu einem Wahrheitswert reduziert werden kann, sollte darauf geachtet werden, daß der 'zentrale' Operator einer der relationalen Operatoren (==, >, <, !=, Und, Oder, Nicht) ist. Die Auswertungsreihenfolge kann beliebig durch Klammersetzung modifiziert werden.

Operatoren

Operator Aktion
+ Addiert zwei Zahlen
- Subtrahiert zwei Zahlen
/ Dividiert zwei Zahlen
* Multipliziert zwei Zahlen
== Vergleicht, ob ein Wert gleich einem anderen ist
> Vergleicht, ob ein Wert größer ist als ein anderer
< Vergleicht, ob ein Wert kleiner ist als ein anderer
!= Vergleicht, ob ein Wert ungleich einem anderen ist
<% Vergleicht zwei Zeichenketten (mit Platzhaltern)
%> Vergleicht zwei Zeichenketten (mit Platzhaltern)
Nicht Negiert einen Wahrheitswert
Und Verknüpft zwei Wahrheitswerte per Konjunktion
Oder Verknüpft zwei Wahrheitswerte per Disjunktion
: Leitet eine Fehlerbehandlung ein

Syntax

  • Um eine Zeichenkette zu verwenden, wird der Text in doppelte Anführungszeichen eingeklammert
  • Um eine Fehlerbehandlung einzuleiten, wird der Fehlerbehandlungsoperator an eine Zeile angehängt und ein Fehlertext
  • Leerzeichen werden weitestgehend ignoriert, d.h. außerhalb von Zeichenketten
  • Zahlen dürfen mit dem unären Minus negiert werden
  • Um zwei Zeichenketten unscharf miteinander zu vergleichen, können entweder <% oder %> verwendet werden. Bei der ersten Form hat der unscharfe Wert auf der linken Seite des Operators zu stehen, bei der letzten From umgekehrt. Erlaubte Platzhalterzeichen sind % für beliebig viele Zeichen oder ? für ein beliebiges Zeichen. Siehe dok/demo.txt.
